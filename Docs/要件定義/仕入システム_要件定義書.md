# 仕入システム 要件定義書

**プロジェクト名**: JASuppliers 仕入システム  
**対象システム**: 委託仕入システム / 委託消化仕入システム / 買取仕入システム  
**作成日**: 2026年5月8日  
**最終更新**: 2026年5月14日  
**バージョン**: 1.1

---

## 1. プロジェクト概要

### 1.1 背景と目的

産直市場（農産物直売所）を運営する小売店において、生産者・仕入先との取引形態は「委託」「委託消化」「買取」の3形態に大別される。これらの仕入形態はそれぞれ精算ロジック・在庫リスク・会計処理が異なるため、統一的に管理できる仕入システムの整備が急務となっている。

本システムは、JAあぐりタウン げんきの郷のような産直市場・農産物直売所を念頭に、3種類の仕入形態を一元管理する仕入管理プラットフォームを構築することを目的とする。

### 1.2 対象業務

| 業務区分 | 説明 |
|----------|------|
| 委託仕入管理 | 生産者から商品を預かり、売れた分だけ精算する仕入管理 |
| 委託消化仕入管理 | 販売実績に基づいて仕入を確定し、未販売在庫はメーカー返品する仕入管理 |
| 買取仕入管理 | 商品を仕入れ時点で所有権を取得し、在庫リスクを自社で負担する仕入管理 |

### 1.3 対象ユーザー

| ユーザー | 役割 |
|----------|------|
| 仕入担当者 | 仕入先登録・発注・入荷・精算処理 |
| 店舗スタッフ | 商品搬入確認・在庫管理・廃棄処理 |
| 生産者・出荷者 | 出荷登録・精算明細確認（出荷者ポータル） |
| 経理担当者 | 精算処理・支払管理・会計連携 |
| システム管理者 | マスタ管理・権限管理・設定変更 |

## 2. 業務要件

### 2.1 委託仕入（委託販売）

#### 2.1.1 業務概要

生産者（出荷者）が商品の所有権を保持したまま、店舗が販売代行を行う。売れた分だけ精算する形態。産直市場・直売所における主流の仕入形態。

```
【生産者（出荷者）】
  ↓ 商品搬入・出荷登録（価格は生産者が決定）
【店舗（産直市場）】
  ↓ 商品受入・陳列（所有権は生産者）
  ↓ 販売（POSスキャン）
  ↓ 売れ残りの回収・廃棄
【精算処理（月次）】
  ↓ 販売金額 - 手数料 = 生産者支払額
  ↓ 精算明細書発行・振込
```

#### 2.1.2 機能要件

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| C-001 | 出荷者登録 | 生産者情報（氏名・住所・口座・品目）の登録・管理 |
| C-002 | 商品登録・バーコード発行 | 品名・価格・生産者情報を含むバーコード（JAN・独自コード）の発行 |
| C-003 | 搬入受付 | 生産者の搬入時の入荷記録（品目・数量・価格） |
| C-004 | 販売実績取込 | POSデータと連携した出荷者別・品目別の販売実績集計 |
| C-005 | 廃棄・回収管理 | 未販売品の廃棄数量・金額の記録 |
| C-006 | 委託精算計算 | 販売金額に手数料率を適用し支払額を算出 |
| C-007 | 精算明細書発行 | 出荷者向け精算明細書（PDF・印刷）の生成 |
| C-008 | 振込データ作成 | 全銀協フォーマットの振込データ出力 |
| C-009 | 出荷者ポータル | 出荷者向け売上照会・精算確認の Web 画面 |
| C-010 | 手数料マスタ管理 | 品目別・出荷者別の手数料率設定 |

#### 2.1.3 業務ルール

| ルール | 内容 |
|--------|------|
| 価格決定 | 生産者（出荷者）が商品価格を決定する |
| 手数料率 | 品目分類・販売形態に応じて設定（例: 野菜15%、加工品20%） |
| 精算サイクル | 月次（翌月末振込が主流）、週次も対応可能 |
| 消費税 | 軽減税率（食料品8%）を適用。税込価格で管理 |
| 廃棄処理 | 廃棄品は精算対象外。廃棄数量を記録し出荷者に報告 |
| 値下げ | 鮮度管理による値下げ処理の記録（値下げ後の販売実績で精算） |
| 返品 | 品質不良品の返品受付・精算控除処理 |

---

### 2.2 委託消化仕入（委託消化）

#### 2.2.1 業務概要

商品が実際に販売された時点で仕入を確定する形態。在庫リスクはサプライヤー（メーカー・農協等）側が負担。百貨店・大型小売店でよく用いられる。産直市場では加工品・ギフト品・アパレルなどに適用。

```
【サプライヤー（メーカー・農協等）】
  ↓ 商品納入（所有権はサプライヤーに留まる）
【店舗】
  ↓ 検品・受入（在庫として積数するが仕入未確定）
  ↓ 販売（POSスキャン）
  ↓ 販売時点で仕入確定
  ↓ 期末に未販売在庫を返品
【精算処理（月次）】
  ↓ 販売確定数量 × 仕入掛率 = 仕入金額
  ↓ 支払・返品処理
```

#### 2.2.2 機能要件

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| CC-001 | 仕入先マスタ管理 | サプライヤー（農協・メーカー等）の情報管理 |
| CC-002 | 商品マスタ管理 | 品目・掛率・上代・下代・在庫単位の登録管理 |
| CC-003 | 納入受付 | 商品の入荷時記録（数量・日付・ロット番号） |
| CC-004 | 委託在庫管理 | サプライヤー別・品目別の委託在庫数量管理 |
| CC-005 | 販売実績取込 | POSデータ連携による販売確定数量の取込 |
| CC-006 | 消化率管理 | 期間別・品目別の消化率（販売率）の算出・監視 |
| CC-007 | 仕入確定処理 | 販売実績に基づく仕入金額の確定処理 |
| CC-008 | 返品処理 | 未販売品のサプライヤーへの返品申請・返品数量管理 |
| CC-009 | 請求照合 | サプライヤーからの請求書と仕入確定金額の照合 |
| CC-010 | 支払管理 | 仕入金額の支払処理・支払明細の管理 |
| CC-011 | 消化レポート | 品目別・サプライヤー別の消化率・在庫推移レポート |

#### 2.2.3 業務ルール

| ルール | 内容 |
|--------|------|
| 仕入確定タイミング | 商品が販売された時点で仕入が確定する |
| 所有権 | 納入時点ではサプライヤーが所有権を保持 |
| 掛率 | 上代に対する掛率（例: 70%掛け = 上代1,000円の場合、仕入700円） |
| 返品条件 | 期末・シーズン終了時に未販売在庫をサプライヤーへ返品 |
| 消費税 | 消費税区分（軽減税率・標準税率）の適用管理 |
| 在庫評価 | 委託在庫は自社在庫と区分して管理（貸借対照表への影響回避） |
| 棚卸 | 月次棚卸時に委託在庫の実棚確認・差異報告 |

---

### 2.3 買取仕入（買取）

#### 2.3.1 業務概要

小売店がメーカー・生産者から商品を購入し、所有権を仕入時点で取得する形態。在庫リスクは自社が負担。精肉・水産物・日配品・米などに広く適用される。

```
【仕入先（メーカー・農協・産地）】
  ↓ 発注（購買発注書の送付）
  ↓ 商品納入
【店舗】
  ↓ 検品・入荷確認（所有権移転）
  ↓ 在庫計上
  ↓ 販売
  ↓ 在庫消化
【精算処理（月次・都度）】
  ↓ 仕入金額の請求・支払
```

#### 2.3.2 機能要件

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| B-001 | 仕入先マスタ管理 | 仕入先（農協・市場・卸・メーカー）の情報管理 |
| B-002 | 商品マスタ管理 | 品目・仕入価格・販売価格・規格・産地の管理 |
| B-003 | 発注管理 | 発注書の作成・承認・送付管理 |
| B-004 | 入荷管理 | 入荷時の検品・受入確認・在庫計上 |
| B-005 | 仕入原価管理 | 仕入価格・仕入原価の管理（FIFO/平均原価法） |
| B-006 | 在庫管理 | 品目別・保管場所別の在庫数量・金額管理 |
| B-007 | 返品・値引処理 | 品質不良・過剰仕入による返品・仕入値引の処理 |
| B-008 | 請求書照合 | 仕入先からの請求書と入荷実績の照合（3-Wayマッチング） |
| B-009 | 支払管理 | 支払条件に基づく支払額の計算・振込処理 |
| B-010 | 発注点管理 | 安全在庫・発注点に基づく自動発注アラート |
| B-011 | 在庫棚卸 | 実棚数量入力・在庫差異の計算・棚卸差損益の計上 |
| B-012 | 仕入分析 | 仕入先別・品目別・期間別の仕入金額・数量分析 |

#### 2.3.3 業務ルール

| ルール | 内容 |
|--------|------|
| 発注承認 | 一定金額以上の発注は上長承認が必要 |
| 入荷検品 | 入荷時に数量・品質の確認を実施。差異は仕入先に即報告 |
| 原価計算 | 移動平均法（または先入先出法）による在庫原価管理 |
| 支払条件 | 月末締め翌月末払いなど仕入先ごとの支払条件を管理 |
| 消費税 | 税区分（食料品8%・その他10%）の適切な管理 |
| ロット管理 | 生鮮品・消費期限管理品のロット番号管理（トレーサビリティ） |
| 価格改定 | 仕入価格変更時の有効日付管理と在庫評価への反映 |

## 3. 共通機能要件

### 3.1 マスタ管理

| 機能 | 説明 |
|------|------|
| 生産者・仕入先マスタ | 取引先情報（基本情報・口座・支払条件）の統合管理 |

| 機能 | 説明 |
|------|------|

### 3.3 帳票出力

| 帳票名 | 形式 | 対象 |
|--------|------|------|
| 精算明細書（委託） | PDF・印刷 | 生産者・出荷者 |
| 消化精算書（委託消化） | PDF・印刷 | サプライヤー |
| 仕入明細書（買取） | PDF・印刷 | 経理担当 |
| 振込依頼書 | 全銀協形式 | 銀行振込 |
| 在庫一覧表 | Excel・PDF | 店舗管理 |
| 仕入先別取引実績 | Excel・PDF | 経営分析 |
| 棚卸差異報告書 | PDF | 棚卸時 |

### 3.4 会計連携

| 機能 | 説明 |
|------|------|
| 仕訳データ出力 | 仕入・支払・在庫調整の会計仕訳データの出力 |
| 消費税区分管理 | 軽減税率（8%）・標準税率（10%）の適切な管理 |
| 勘定科目マッピング | 仕入形態・商品分類に基づく勘定科目の自動適用 |

### 3.5 セキュリティ要件

| 要件 | 内容 |
|------|------|
| 認証 | ユーザーID・パスワード認証（パスワードポリシー設定必須） |
| 権限管理 | ロールベースアクセス制御（仕入担当・経理・管理者等） |
| 操作ログ | 全操作の監査ログの記録・保管（7年間） |
| データ暗号化 | 口座情報等の個人情報・機密情報の暗号化保存 |
| バックアップ | 日次バックアップ・世代管理（30日間保持） |

## 4. 非機能要件

### 4.1 パフォーマンス要件

| 項目 | 要件 |
|------|------|
| レスポンス時間 | 画面操作の応答時間：3秒以内（通常処理） |
| バッチ処理 | 日次精算バッチ：深夜2時間以内に完了 |
| 同時接続 | 20ユーザーの同時利用に耐えること |
| データ保持 | 10年分の取引データを保持・検索可能であること |

### 4.2 可用性要件

| 項目 | 要件 |
|------|------|
| 稼働時間 | 店舗営業時間（7:00〜21:00）は99%以上の稼働率 |
| 計画停止 | 月1回の深夜メンテナンスウィンドウを設定可能 |
| 障害復旧 | 障害発生から4時間以内に復旧（RTO: 4時間） |
| データ復旧 | 最大1日分のデータ損失を許容（RPO: 24時間） |

### 4.3 拡張性要件

| 項目 | 要件 |
|------|------|
| 店舗拡張 | 複数店舗への拡張に対応できる設計 |
| API連携 | 外部POSシステム・会計システムとのAPI連携 |
| モバイル対応 | タブレット・スマートフォンによる入荷検品・棚卸への対応 |

  POS/レジシステム ←→ 仕入システム ←→ 会計システム

### 5.2 システム化対象外（手動対応）

- 生産者との口頭による出荷調整・コミュニケーション
- 商品の品質検査・農薬検査（専門機関への依頼）
- 消費者へのクレーム対応・返金処理（POS側で対応）

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| 委託（販売委託） | 生産者が商品の所有権を保持したまま、店舗が販売代行を行う形態 |
| 委託消化（委託消化仕入） | 商品が販売された時点で仕入を確定させ、未販売品はサプライヤーへ返品する仕入形態 |
| 買取 | 仕入時点で所有権が店舗に移転し、在庫リスクを店舗が負担する仕入形態 |
| 出荷者 | 委託販売において商品を店舗に出荷・搬入する生産者・農家 |
| 仕入先 | 商品を供給するサプライヤー（農協・メーカー・卸売業者・生産者） |
| 手数料率 | 委託販売における店舗の取り分（売上に対する割合） |
| 掛率 | 委託消化における仕入価格の算出比率（上代に対する割合） |
| 精算 | 出荷者・サプライヤーへの売上・仕入金額の確定と支払 |
| 消化率 | 委託消化における入荷数に対する販売数の比率 |
| トレーサビリティ | 産地・生産者・出荷経路を追跡できる仕組み |
| 軽減税率 | 食料品等に適用される消費税率8%（通常税率10%との区分管理が必要） |

## 7. 制約事項・前提条件

| 項目 | 内容 |
|------|------|
| 法的要件 | 農薬取締法・食品衛生法・消費税法の遵守 |
| 帳票保存 | 仕入関係書類の7年間保存（法人税法） |
| 個人情報保護 | 生産者・仕入先の個人情報の適切な管理（個人情報保護法） |
| POSシステム依存 | 販売実績の取込は既存POSシステムのデータ仕様に依存 |
| 会計システム連携 | 使用中の会計システムとのデータ形式の整合性確認が必要 |
| 消費税区分 | 食料品（軽減税率8%）と一般商品（標準税率10%）の区分管理が必須 |
| 開発フレームワーク | PHP 8.5 / Laravel 13.x を初期標準とする |
| 標準データベース | PostgreSQL 18.x を初期標準とする |
| DB切替 | Laravelの `.env` / `config/database.php` により接続先DBを切替可能にする |
| ソース管理 | GitHub `https://github.com/KoukiKimura/JASuppliers.git` を公式リポジトリとする |
| 環境反映 | 各環境ではGitHubから最新ソースを取得し、ローカル環境で実行する |
| ローカル実行 | WSL2 / Docker Desktop を必須にしない。WindowsではLaravel Herd + PostgreSQL Windows版を標準候補とする |
| Mock確認 | 初期データSeederを用意し、相手環境でローカル実行して確認できる構成とする |
