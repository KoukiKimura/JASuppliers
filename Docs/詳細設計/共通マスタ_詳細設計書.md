**プロジェクト名**: JASuppliers 仕入システム  
**作成日**: 2026年5月14日  
**最終更新**: 2026年5月14日  
**バージョン**: 1.0  
**対象要件**: 3.1、3.3、3.5、3.6、C-001、C-002、C-010、CC-001、CC-002、B-001、B-002、B-009  

---

## 1. 目的

仕入3形態で共通利用する取引先、商品、店舗、税区分、手数料率、掛率、支払条件、会計マッピングの詳細仕様を定義する。

---

## 2. 共通設計ルール

| 項目 | 仕様 |
|------|------|
| 主キー | `id bigserial` を標準とする |
| 業務キー | `*_code` を保持し、画面・CSV連携で利用する |
| 論理削除 | 業務マスタは `deleted_at` を保持する |
| 監査列 | `created_at`、`updated_at`、`created_by`、`updated_by` を保持する |
| 楽観ロック | 更新頻度が高いマスタは `lock_version` を保持する |
| 文字コード | UTF-8 |
| 日付 | `date` または ISO 8601 datetime |
| 金額 | 日本円、整数、税込基準 |
| 暗号化 | 口座情報、個人情報、連絡先の一部を暗号化対象とする |

## 6. マスタ登録・更新処理

1. 入力値を検証する。
2. 業務キーの重複を確認する。
3. 有効期間を持つ料率・税率・掛率は期間重複を確認する。
4. 口座情報・個人情報は暗号化して保存する。
5. 登録・更新・論理削除を行う。
6. 操作ログを `audit_logs` に記録する。
7. POS取込や精算で参照中のマスタは、削除ではなく無効化を優先する。

## 7. バリデーション

| 対象 | ルール |
|------|--------|
| 取引先コード | 必須、64文字以内、一意 |
| 取引先種別 | 必須、登録済み種別 |
| 口座情報 | 振込対象の場合は必須 |
| 商品コード | 必須、64文字以内、一意 |
| 仕入形態 | 必須、`CONSIGNMENT` / `CONSUMPTION` / `PURCHASE` |
| 税区分 | 必須、8%または10% |
| 手数料率・掛率 | 0以上100以下、有効期間重複不可 |
| 支払条件 | 締日、支払月、支払日が必須 |

## 9. テスト観点

- 同一コードの重複登録ができないこと。
- 論理削除済みマスタが通常検索に表示されないこと。
- 手数料率・掛率の有効期間重複が防止されること。
- 暗号化対象項目が平文で保存されないこと。
- 権限のない利用者がマスタ管理APIを実行できないこと。
- SeederでMock確認に必要な初期データが投入されること。
