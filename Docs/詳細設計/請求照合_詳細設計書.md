# 請求照合 詳細設計書

**プロジェクト名**: JASuppliers 仕入システム
**作成日**: 2026年5月14日
**最終更新**: 2026年5月14日
**バージョン**: 1.0
**対象要件**: CC-009、B-008

---

## 1. 目的

委託消化仕入と買取仕入において、仕入確定・入荷実績と仕入先請求書を照合し、差額を検出・承認して支払へ連携する詳細仕様を定義する。

---

## 2. 対象範囲

| 区分 | 対象 |
|------|------|
| 委託消化 | 仕入確定明細と請求書の照合 |
| 買取 | 入荷明細と請求書の3-Wayマッチング |
| 初期入力 | 画面入力または汎用CSV取込 |
| 対象外 | OCR取込、仕入先ポータル、会計ソフト別専用連携 |

---

## 3. 要件対応

| 機能ID | 詳細設計での対応 |
|--------|------------------|
| CC-009 | サプライヤー請求書と委託消化仕入確定金額の照合 |
| B-008 | 仕入先請求書と発注・入荷実績の3-Wayマッチング |

---

## 4. テーブル定義

## 7. 照合処理

1. 請求書ヘッダと明細を登録する。
2. 対象年月・仕入先・仕入形態からシステム側明細を抽出する。
3. 商品、数量、金額で照合する。
4. 差額が0円の場合は `RECONCILED` とする。
5. 差額がある場合は `DIFFERENCE` とし、理由入力と承認を要求する。
6. 承認済み請求書を支払対象へ連携する。

---

## 8. バリデーション

| 対象 | ルール |
|------|--------|
| 請求書番号 | 仕入先内で一意 |
| 請求金額 | 0以上 |
| 対象年月 | 精算または入荷実績が存在する年月 |
| 差額理由 | 差額がある場合は必須 |
| 差額承認 | 管理者権限が必須 |
| 支払連携 | 未照合または未承認差額がある請求書は支払対象外 |

---

## 9. テスト観点

- 請求書を登録できること。
- 差額なしの請求書が照合済みになること。
- 差額ありの請求書が差額状態になること。
- 差額理由なしでは承認できないこと。
- 未承認差額が支払対象にならないこと。
- 取消時に監査ログが記録されること。
