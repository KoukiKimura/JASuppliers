# 委託仕入 詳細設計書

**プロジェクト名**: JASuppliers 仕入システム  
**作成日**: 2026年5月14日  
**最終更新**: 2026年5月14日  
**バージョン**: 1.0  
**対象要件**: C-001、C-002、C-003、C-004、C-005、C-006、C-007、C-008、C-010  

---

## 1. 目的

委託仕入における出荷者、商品、搬入、販売実績、廃棄・回収、月次精算、精算明細、全銀協形式CSV出力の詳細仕様を定義する。

C-009 出荷者ポータルは後続フェーズのため、本書では管理者・仕入担当者・店舗スタッフ・経理担当者が利用する機能を対象とする。

---

## 2. 業務ルール

| 項目 | 仕様 |
|------|------|
| 所有権 | 商品所有権は出荷者に残る |
| 価格決定 | 出荷者が決定した税込売価を登録する |
| 精算単位 | 出荷者、店舗、対象年月 |
| 精算サイクル | 月次 |
| 支払額 | 販売税込金額 - 手数料額 + 調整額 |
| 手数料端数 | 1円未満四捨五入 |
| 廃棄・回収 | 精算対象外として数量・理由を記録する |
| 返品 | POS返品は負数の販売実績として精算控除する |
| 締め後訂正 | 元明細を直接上書きせず、訂正取引または翌月調整とする |

## 7. 月次精算処理

### 7.1 抽出条件

| 項目 | 条件 |
|------|------|
| 対象仕入形態 | `pos_sales_lines.procurement_type = CONSIGNMENT` |
| 対象期間 | `business_date` が対象年月内 |
| 締め状態 | 未締め明細のみ |
| 対象外 | 廃棄・回収、取込エラー、取消済み明細 |

### 7.2 計算式

```text
販売税込金額 = sum(pos_sales_lines.net_amount)
手数料額 = round(販売税込金額 * 手数料率 / 100)
支払額 = 販売税込金額 - 手数料額 + 調整額
```

### 7.3 処理手順

1. 対象年月、店舗、出荷者を指定する。
2. 委託POS販売実績を集計する。
3. 商品カテゴリ・出荷者別の手数料率を取得する。
4. 精算明細を作成する。
5. 担当者が確認する。
6. 精算を締める。
7. 精算明細PDFと全銀協形式CSVを出力する。
8. 対象POS明細を締め済みに更新する。

---

## 8. バリデーション

| 対象 | ルール |
|------|--------|
| 搬入日 | 必須、未来日不可 |
| 出荷者 | 委託出荷者フラグが有効 |
| 商品 | 仕入形態が `CONSIGNMENT` |
| 数量 | 0より大きい |
| 売価 | 0以上 |
| 精算締め | 対象年月に未解決POS取込エラーがないこと |
| 締め解除 | 管理者権限かつ理由入力必須 |

---

## 9. テスト観点

- 搬入登録、確定、取消ができること。
- 廃棄・回収が精算対象外になること。
- POS返品が支払額から控除されること。
- 手数料率が有効期間に従って適用されること。
- 精算締め後のPOS再取込が直接更新されないこと。
- 精算明細PDFと全銀協形式CSVが出力されること。
