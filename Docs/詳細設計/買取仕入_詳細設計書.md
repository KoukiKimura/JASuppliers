# 買取仕入 詳細設計書

**プロジェクト名**: JASuppliers 仕入システム  
**作成日**: 2026年5月14日  
**最終更新**: 2026年5月14日  
**バージョン**: 1.0  
**対象要件**: B-001〜B-012  

---

## 1. 目的

買取仕入における仕入先、商品、発注、承認、入荷、原価、在庫、返品・値引、請求照合、支払、発注点、棚卸、分析の詳細仕様を定義する。

---

## 2. 業務ルール

| 項目 | 仕様 |
|------|------|
| 所有権 | 入荷確定時点で店舗へ移転する |
| 在庫区分 | 自社在庫として数量・金額を管理する |
| 原価計算 | 初期は移動平均法を標準とし、先入先出法は詳細実装時の拡張候補とする |
| 発注承認 | 一定金額以上の発注は承認を必須とする |
| ロット管理 | 商品設定によりロット番号・消費期限を管理する |
| 請求照合 | 発注、入荷、請求の3-Wayマッチングを行う |
| 支払 | 支払条件に基づき支払予定を作成する |
| 棚卸 | 実棚数と帳簿数の差異を在庫調整として記録する |

## 7. 入荷・在庫計上処理

### 7.1 処理手順

1. 発注を登録する。
2. 金額条件に応じて承認申請する。
3. 承認後、発注状態を `ORDERED` にする。
4. 入荷検品を登録する。
5. 入荷確定時に在庫を増加する。
6. 移動平均原価を再計算する。
7. 請求書と入荷実績を照合する。
8. 支払予定を作成する。

### 7.2 移動平均原価

```text
新平均原価 = round((既存在庫金額 + 入荷金額) / (既存在庫数量 + 入荷数量))
新存在庫金額 = 新平均原価 * 新存在庫数量
```

返品・棚卸差異は `inventory_movements` に記録し、在庫数量・在庫金額へ反映する。

## 8. 発注点アラート

| 条件 | 処理 |
|------|------|
| `stock_quantity <= reorder_point_quantity` | 発注候補として表示 |
| 安全在庫未満 | 警告表示 |
| 発注済未入荷あり | 重複発注警告 |

---

## 9. バリデーション

| 対象 | ルール |
|------|--------|
| 仕入先 | 買取仕入先フラグが有効 |
| 商品 | 仕入形態が `PURCHASE` |
| 発注数量 | 0より大きい |
| 発注単価 | 0以上 |
| 入荷数量 | 発注数量を超える場合は警告 |
| 不良数量 | 入荷数量以下 |
| ロット | ロット管理対象商品の場合は必須 |
| 請求照合 | 差額がある場合は理由入力必須 |

---

## 10. テスト観点

- 発注登録、承認、却下、取消ができること。
- 入荷確定で在庫数量・在庫金額が増加すること。
- 移動平均原価が再計算されること。
- ロット管理対象商品でロット未入力がエラーになること。
- 請求書と入荷実績の差額が検出されること。
- 棚卸差異が在庫調整として記録されること。
- 発注点を下回る商品が候補表示されること。
