# 仕入システム 認証権限 詳細設計書

**プロジェクト名**: JASuppliers 仕入システム
**作成日**: 2026年5月14日
**最終更新**: 2026年5月14日
**バージョン**: 1.0
**対象要件**: 3.6、共通基盤

---

## 1. 目的

Laravel標準認証を基盤に、管理者、仕入担当者、店舗スタッフ、経理担当者のロール別権限を定義する。初期リリースでは社内利用を前提とし、出荷者ポータル向けの外部ユーザー認証は後続フェーズで扱う。

## 2. 対象範囲

| 区分 | 対象 |
|---|---|
| 対象機能 | ログイン、ログアウト、ログインユーザー取得、ユーザー管理、ロール管理、操作ログ参照 |
| 対象画面 | ログイン画面、ユーザー管理画面、ロール管理画面、操作ログ画面 |
| 対象外 | 出荷者ポータル認証、多要素認証、外部IdP連携 |

## 3. ロール・権限設計

| ロール | 主な利用者 | 許可する操作 |
|---|---|---|
| 管理者 | システム管理者 | 全機能、マスタ管理、ユーザー管理、ロール管理、操作ログ参照 |
| 仕入担当者 | 仕入業務担当 | 出荷者・商品マスタ参照更新、入荷検品、委託・買取仕入、POS取込、精算確認 |
| 店舗スタッフ | 店舗担当者 | 入荷予定参照、検品登録、在庫・販売集計参照 |
| 経理担当者 | 経理担当 | 精算確定、請求照合、支払データ出力、会計連携CSV出力 |

## 5. API仕様

| API | Method | 概要 | 主な権限 |
|---|---|---|---|
| `/api/auth/login` | POST | ログイン | 未認証 |
| `/api/auth/logout` | POST | ログアウト | 認証済み |
| `/api/auth/me` | GET | ログインユーザーと権限取得 | 認証済み |
| `/api/admin/users` | GET | ユーザー一覧取得 | user.view |
| `/api/admin/users` | POST | ユーザー登録 | user.create |
| `/api/admin/users/{id}` | PUT | ユーザー更新 | user.update |
| `/api/admin/roles` | GET | ロール一覧取得 | role.view |
| `/api/admin/roles/{id}/permissions` | PUT | ロール権限更新 | role.update |
| `/api/audit-logs` | GET | 操作ログ検索 | audit.view |

### 5.1 ログインリクエスト

```json
{
  "email": "buyer@example.local",
  "password": "password"
}
```

### 5.2 ログインレスポンス

```json
{
  "user": {
    "id": 1,
    "name": "仕入担当者",
    "email": "buyer@example.local",
    "roles": ["buyer"],
    "permissions": ["supplier.view", "supplier.update", "receiving.create"]
  }
}
```

## 6. 画面仕様

| 画面ID | 画面名 | 主な項目 | 主な操作 |
|---|---|---|---|
| AUTH-001 | ログイン | メールアドレス、パスワード | ログイン |
| ADM-001 | ユーザー管理 | ユーザー名、メールアドレス、状態、ロール | 検索、登録、更新、停止 |
| ADM-002 | ロール管理 | ロール、権限一覧 | 権限付与、権限解除 |
| AUD-001 | 操作ログ | 操作日時、ユーザー、操作種別、対象 | 検索、詳細表示 |

## 7. 認証・権限判定処理

1. ログイン時にメールアドレスとパスワードを検証する。
2. ユーザー状態が `active` 以外の場合はログイン不可とする。
3. 認証成功時に `last_login_at` を更新する。
4. APIリクエストごとに認証済みユーザーを取得する。
5. ルートまたはControllerで要求権限コードを判定する。
6. 権限不足の場合はHTTP 403を返却する。
7. 登録、更新、削除、確定、出力の操作は `audit_logs` に記録する。

## 8. バリデーション

| 対象 | ルール |
|---|---|
| email | 必須、メール形式、255文字以内、ユーザー登録時はユニーク |
| password | 必須、8文字以上、保存時はLaravelのHashでハッシュ化 |
| status | active、inactive、locked のいずれか |
| role_id | 存在するroles.id |
| permission_id | 存在するpermissions.id |
| audit_logs検索条件 | 日付範囲は開始日が終了日以前 |
