# 委託消化仕入 詳細設計書

**プロジェクト名**: JASuppliers 仕入システム  
**作成日**: 2026年5月14日  
**最終更新**: 2026年5月14日  
**バージョン**: 1.0  
**対象要件**: CC-001〜CC-011  

---

## 1. 目的

委託消化仕入における仕入先、商品、納入、委託在庫、販売実績取込、消化率、仕入確定、返品、請求照合、支払管理、消化レポートの詳細仕様を定義する。

---

## 2. 業務ルール

| 項目 | 仕様 |
|------|------|
| 所有権 | 納入時点ではサプライヤーが保持する |
| 仕入確定 | 商品販売時点で販売実績に基づき確定する |
| 仕入金額 | 販売確定数量 × 上代 × 掛率 |
| 在庫区分 | 自社在庫と区別して委託消化在庫として管理する |
| 未販売在庫 | 期末またはシーズン終了時に返品対象とする |
| 消化率 | 販売数量 ÷ 納入数量 |
| 請求照合 | 仕入確定金額とサプライヤー請求書を照合する |
| 締め後訂正 | 訂正取引または翌月調整で扱う |

## 7. 仕入確定処理

### 7.1 抽出条件

| 項目 | 条件 |
|------|------|
| 対象仕入形態 | `pos_sales_lines.procurement_type = CONSUMPTION` |
| 対象期間 | 対象年月内の販売実績 |
| 対象明細 | 未締め、取込エラーなし |
| 返品 | 負数数量・負数金額として控除 |

### 7.2 計算式

```text
販売確定数量 = sum(pos_sales_lines.quantity)
仕入単価 = 上代税込単価 * 掛率 / 100
仕入金額 = round(販売確定数量 * 仕入単価)
消化率 = 販売数量 / 納入数量 * 100
```

### 7.3 処理手順

1. 対象年月、店舗、サプライヤーを指定する。
2. POS販売実績を集計する。
3. 納入実績・委託在庫残高を取得する。
4. 掛率を適用して仕入確定明細を作成する。
5. 委託在庫の販売数量・残数量を更新する。
6. 請求書登録後、仕入確定金額と照合する。
7. 支払対象を確定する。

---

## 8. バリデーション

| 対象 | ルール |
|------|--------|
| サプライヤー | 委託消化仕入先フラグが有効 |
| 商品 | 仕入形態が `CONSUMPTION` |
| 納入数量 | 0より大きい |
| 掛率 | 0より大きく100以下 |
| 返品数量 | 残数量以下 |
| 仕入確定 | 未解決POS取込エラーがないこと |
| 請求照合 | 差額がある場合は理由入力必須 |

---

## 9. テスト観点

- 納入確定により委託在庫が増加すること。
- POS販売実績により販売数量・残数量が更新されること。
- 掛率に基づき仕入金額が計算されること。
- 返品により残数量が減少すること。
- 請求金額と仕入確定金額の差額が検出されること。
- 消化レポートで消化率が確認できること。
