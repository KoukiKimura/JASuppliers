# 仕入システム ロードマップ

**プロジェクト名**: JASuppliers 仕入システム  
**作成日**: 2026年5月14日  
**バージョン**: 1.2

---

## 1. ロードマップ方針

| 項目 | 方針 |
|------|------|
| 初期リリース対象 | 委託仕入・委託消化仕入・買取仕入の3形態すべて |
| Mock確認 | Windowsローカル環境で構築・実行する方式を標準とする |
| POS連携 | 既存サンプル未入手のため、初期は想定CSV形式で設計・実装する |
| 出荷者ポータル | 後続フェーズで実装する |
| デプロイ | 初期リリース開発後に別途検討する |
| 店舗範囲 | 初期リリースは単店舗運用。店舗コードを保持し、将来の複数店舗化に備える |
| 精算サイクル | 初期リリースは月次精算。週次精算は後続フェーズで検討する |

## 3. マイルストーン

| フェーズ | 目的 | 主な成果物 | 完了条件 |
|----------|------|------------|----------|
| Phase 0 | 企画・前提整理 | 要件定義、ロードマップ、POSサンプルCSV、ローカル実行手順 | GitHubに資料一式が反映されている |
| Phase 1 | Laravel基盤構築 | Laravel 13、PostgreSQL接続、認証、権限、共通レイアウト | Windowsローカルで起動できる |
| Phase 2 | 共通マスタ・POS取込 | 取引先、取引先種別、商品、店舗、税区分、手数料率、掛率、POS取込ステージング | サンプルCSVを取り込める |
| Phase 3 | 委託仕入実装 | 搬入、販売実績、廃棄、委託精算、精算明細 | 委託仕入の月次精算をMockデータで確認できる |
| Phase 4 | 委託消化仕入実装 | 納入、委託在庫、消化率、仕入確定、返品、支払 | 委託消化の仕入確定をMockデータで確認できる |
| Phase 5 | 買取仕入実装 | 発注、入荷、在庫、原価、請求照合、支払 | 買取仕入の入荷から支払まで確認できる |
| Phase 6 | 初期リリース検証 | 結合確認、帳票、会計CSV・全銀協形式CSV、Seeder、ローカル確認手順 | 各ローカル環境で初期リリース範囲を確認できる |

## 5. リスクと対応方針

| リスク | 内容 | 対応 |
|--------|------|------|
| POS仕様未確定 | 実POSのデータ形式が不明 | 初期は想定CSVで実装し、ステージング取込で差分吸収する |
| 3形態同時実装 | 初期リリース範囲が広い | 共通マスタ・POS取込・精算基盤を先に作り、各形態の差分を段階的に実装する |
| 金額計算の差異 | 税、値下げ、返品、端数処理で誤差が出る | 基本設計で計算単位、丸め、締め後訂正を明文化する |
| ローカル環境差異 | 確認者ごとにWindows環境が異なる | Laravel Herd + PostgreSQL Windows版を標準手順とし、Seederで同一データを再現する |
| 出荷者ポータル後回し | 生産者向け確認が初期リリースに含まれない | 初期は管理者・担当者画面で精算明細を確認できるようにする |
| 複数店舗化 | 初期から複数店舗業務を作り込むと範囲が膨らむ | 初期は単店舗運用とし、店舗コード保持により拡張余地を確保する |
